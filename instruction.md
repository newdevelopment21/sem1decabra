## Что такое git.

`Git` - это консольная утилита, для отслеживания и ведения истории изменения файлов, в проекте. Чаще всего его используют для кода, но можно и для других файлов. Например, для картинок - полезно для дизайнеров.

С помощью Git-a можно откатить свой проект до более старой версии, сравнивать, анализировать или сливать свои изменения в репозиторий.

Репозиторием называют хранилище кода и историю его изменений. Git работает локально и все ваши репозитории хранятся в определенных папках на жестком диске.

Так же репозитории можно хранить и в интернете. Обычно для этого используют три сервиса:

`GitHub`

`Bitbucket`

`GitLab`

Каждая точка сохранения проекта носит название коммит (commit). У каждого commit-a есть hash (уникальный id) и комментарий. Из таких commit-ов собирается ветка. Ветка - это история изменений. У каждой ветки есть свое название. Репозиторий может содержать в себе несколько веток, которые создаются из других веток или вливаются в них.

и только после этого ввести в терминале команду:

Команда git commit делает для проекта снимок текущего состояния изменений, добавленных в раздел проиндексированных файлов. Это снимки актуальных состояний, а не отличия.

Эта команда  по умолчанию откроет текстовый редактор с предложением ввести комментарий к коммиту. После ввода комментария сохраните файл и закройте текстовый редактор, чтобы выполнить коммит.    

Есть разные флаги для git commit,чтобы облегчить жизнь программистам, например 

    git commit -m "информация о изменениях"

позволяет не обращаться к редактору, а напрямую вводить сообщение не используя редактор.
 -m - это сокращение от "message"

    git commit -a

git commit -a это команда add в соеращенном виде
Эта команда сработее только если в данном файле ранее уже была хоть раз использована команда 
git add <имя файла> .

следующая команда:

    git commit -am 

Данная команда позволяет обьединить два действия в одно, но я должна быть на 100 процентов уверена что данные внесенные в память таким способом мне уже не понадобится менять, так как для этого способа характерна фиксация без возможности вносить изменения.

    git commit --amend

Этот параметр добавляет команде commit новый уровень функциональности. При передаче этого параметра будет изменен последний коммит. 
## **git commit --amend эта комада Вместо создания нового коммита проиндексированные изменения будут добавлены в предыдущий коммит. Кроме того, эта команда откроет настроенный текстовый редактор системы с предложением изменить указанный ранее комментарий к коммиту.**


## Журнал коммитов. Что такое хэш, (хэш код, хэш сумма, индекс ) и как его найти, для чего использовать.

В каждом репозитории ведется журнал коммитов, для того чтобы посмотреть его , нужно ввести команду :

    git log

Данная команда выведет журнал, список коммитов, там будет информация о том кто, когда его создал и само содержание коммита. А также номер коммита. Номер коммита еще называют хэш сумма, хэш, индекс, его используют для перемещения по журналу, для того чтобы быстро найти и открыть нужный раздел. 
Команда git log имеет несколько флагов, например:

    git log --oneline   вывод в одну строку

    git log --all  выведет все индексы если до этого   переключались по веткам или голова HEAD оторвалась от master

    git log --oneline --all  комбинация сочетает два флага: выведет все коммиты в одну строчку

## Как посмотреть и сравнить изменеия между последним изменением и последним сохранением или между двумя файлами возможно даже на разных ветках?

чтобы посмотреть изменения нужно ввести в терминале команду 

    git diff

Эту команду можно дополнить указанием на конкретный  (хэш) 

    git diff 9a72e56 d931ae4

такая функция вызовет два конкретных коммита с этими хэш суммами и сравнит эти снимки между собой, подсветив изменения. Очень удобно!

## HEAD отделился от master

Важно обсудить состояние **detached HEAD**, или состояние открепленного указателя HEAD. 
 HEAD — это указатель на текущий снимок в Git. По сути , команда git checkout просто обновляет указатель HEAD, чтобы он ссылался на указанную ветку или коммит. Когда HEAD указывает на ветку, Git молчит, **но при попытке переключиться на коммит система переходит в состояние detached HEAD (открепленный указатель HEAD).**  Это сообщение предупреждает о том, что вся текущая работа «откреплена» от остальной части вашего проекта. **Если вы начнете разрабатывать функцию, находясь в состоянии открепленного указателя HEAD, у вас не будет ветки, которая позволила бы вам вернуться к этой функции.**
  Когда вы неизбежно переключитесь на другую ветку (например, чтобы слить код своей функции), вы уже никак не сможете сослаться на свою функцию, поэтому в состоянии **detached HEAD** вернуть голову можно с помощью команды :

    git checkout master
